Import

Prepare an Integration Server

  1. Assuming you are using a Windows platform, start up your Integration Server by typing the following command into the ACE Command Console:
    IntegrationServer --name MyServer --work-dir C:\MyServer
    Where C:\MyServer will be the working directory for your server.
  2. Open up your preferred web browser and type this link into the search bar:
    localhost:7600/apidocs/
  3. After a moment you should be presented with some comprehensive documentation on the Admin REST Api, specifically for your standalone server.
  4. Now, create and launch an Integration Node in a seperate ACE Command Console. By default the port for a node is 4414, which is what this tutorial will assume your port is.
  5. Once created and launched, open up a new tab in your web browser and connect to:
    localhost:4414/apidocs/
  6. You should be presented with a page of documentation again. However, there are a few key differences in the actions permitted by the Api when dealing with a node as opposed to a standalone server.
  7. Finally, deploy SimpleApp.bar to your standalone server.

Deploy

Follow these steps to complete the tutorial

Standalone server:

  1. Open the tab with the documentation for your standalone server.
    On the left hand side you should see all the available actions through the api. In this part of the tutorial we are going to GET the details for this server, DEPLOY a simple application to the server, interact with this application on the server and then shut down the server; all from the documentation.
  2. Click on the GET / tab in the list of actions. The body of the docs should change to present you with all the details on this action. You'll be presented with the description of the action, it's parameters (and their structure), example requests from various languages (curl, python etc.) and examples of the standard responses.
    Go back to the top of this page and select the 'Try it' tab.
  3. Here you can run the action against your server as an example. Don't change anything for now, go ahead and click 'Send' in the bottom-right. After a brief pause you should be presented with the real request to, and response from, your server.
  4. Scrolling down and inspecting the JSON response, you should see a few sections:
    - The name and type of your server at the very top
    - A section called 'actions' which shows you all the valid actions on the server itself
    - Below sections such as 'applications', 'policies' and 'restApis' with details on whether they contain any children.
  5. If you go down to GET /applications on the left and try that out the response should display the name 'SimpleApp'
  6. Run a GET /applications/{application} to get more details about a deployed application. Change the parameters of the request to 'SimpleApp'.
  7. Finally, navigate to POST /shutdown and trigger a shutdown event to stop the server. Refreshing the page or checking the ACE Command Console will confirm whether the server has actually terminated or not.

Integration Node:

  1. In this section we will assume you have completed the above tutorial for the standalone server.
  2. Open the tab with the documentation for your integration node.
    On the left hand side you should see all the available actions through the api. In this part of the tutorial we are going to create a server under the node, fetch the server properties and then remove the server all from within this documentation.
  3. Navigate to GET /servers and Try it out. After sending the request to the Node you should recieve a response, either showing that there are no child servers or displaying details about any existing ones.
  4. Now, navigate to POST /servers and proceed to the 'Try it' section. Generate an input for the Body field and then simply change the name to 'myNodeServer' for now. Click 'Send' and run another GET /servers to see the updated server list.
  5. Assuming the above steps have gone well, navigate to GET /servers/{server} and the 'Try it' section. Copy the name of your newly created server into the 'Path' field and click send. The response at the bottom of the page should then present all the properties of the newly created server to you.
  6. Then, navigate to POST /servers/{server}/stop then under 'Try it' input the name of your new server and click 'Send' to stop the server. This may take a while, wait patiently.
  7. Finally, navigate to DELETE /servers/{server} then under 'Try it' input the name of your stopped server and click 'Send' to delete the server.
  8. Run one last get request against the Node to check that the server doesn't exist anymore.
